07 — Governance Rules
Rule 1 — No symbols
No module carries a geometric mark, icon, or abstract symbol. Ever. The name is the identity. The number is the position. Adding a symbol would fragment the system into separate products.
Rule 2 — No color differentiation
All three modules use the same color system. Teal (#f5f4f0 on #090d18) applies to layer numbers only — the same teal for all three. Different colors per module would create separate brand identities.
Rule 3 — Devanagari is always equal
Devanagari and English are co-equal layers. Devanagari never appears smaller than 16px. It is never a subtitle, caption, or translation. It carries the name; it does not explain it.
Rule 4 — The number is structural
01, 02, 03 are infrastructure layer identifiers — not rankings, not order of importance. They communicate: these are parts of a numbered system, like network layers or technical protocol stacks.
Rule 5 — Always left-aligned
Module lockups are always left-aligned. Never centered. Centered type reads as designed. Left-aligned type reads as structured. Infrastructure is structured, not designed.
Rule 6 — Parent always present
"AyuVinc" always appears in the full lockup's descriptor line — at minimum legible opacity, but always present. A module is never seen fully divorced from the parent system.
08 — Phase 3 Summary
Module System Architecture · Approved Direction
One system · Three layers · No symbols · Layer numbers as the only differentiator
The module identity system is typographic throughout. Space Grotesk 300 for English. Noto Sans Devanagari 300 for the cultural continuity layer. DM Sans 400 for all labels and metadata. The three modules share identical treatment — the name, the Devanagari form, and the layer number are the complete identity. Nothing else is needed. The system reads as institutional infrastructure because it behaves like institutional infrastructure: consistent, numbered, restrained.
